# Speedup

Quick status briefing that reads the project backlog and presents the most actionable items.

## Instructions

1. Read `Plans/Project_Status.md`
2. Identify and present:
   - Any items marked as high priority, urgent, or "next"
   - Items at the top of the backlog (first 3-5 items)
   - Any blocked items that need attention
3. Present the items in a concise, scannable format
4. If there are items that seem quick to knock out, mention those
5. Keep the response brief — this is meant to get the user oriented quickly, not provide exhaustive detail

## Output Format

Use a format like:

**Immediate Items:**
- Item 1
- Item 2
- Item 3

**Quick Wins:** (if any obvious ones exist)
- Small task that could be done fast

**Blocked/Needs Attention:** (if any)
- Item needing resolution
