1. Getting Started

What is MILESTONE?

MILESTONE is a construction activity tracking and work package generation system built for Summit Constructors. Field engineers use it to track progress on industrial construction activities (Receiving, Fabrication, Assembly, Connections, etc.) and generate work packages for field crews. The system integrates with P6 Primavera for schedule management.

Before You Begin

Before using MILESTONE:

  • Admin setup required — A system administrator must add your Windows username to the MILESTONE user database
  • No login needed — MILESTONE automatically detects your Windows username
  • First sync — On first launch, you'll need to sync with the central Azure database to download project data (or import or create activities if none exist yet for your project). If you import or create new records, they will be assigned to you initially — but you can reassign them if desired.

3. Progress Module

Overview

The Progress Module is where you track construction activity completion. This is the primary workspace for field engineers entering daily progress and reviewing assigned work.

Screenshot: Progress module full view (progress-full-view.png)

Typical Workflow

The standard daily workflow is:

  1. Filter — Use the sidebar to filter to your records or specific criteria
  2. Edit — Update percent complete and other fields in the grid
  3. Sync — Push your changes to Azure and pull updates from others

Layout

The Progress view has three main areas:

  • Left Sidebar — Quick filter buttons and controls
  • Main Grid — Activity data with toolbar above
  • Footer — Record count and metadata error indicator

Summary Row

A summary row at the bottom of the grid displays totals for key numeric columns:

  • Quantity — Sum of all quantities
  • EarnQtyEntry — Sum of earned quantities
  • BudgetMHs — Sum of budgeted man-hours
  • EarnMHsCalc — Sum of earned man-hours
  • ClientBudget — Sum of client budget values

The summary row updates automatically when you edit values or apply filters, showing totals for the currently visible records only.

  • Budget — Total budgeted man-hours
  • Earned — Total earned man-hours based on percent complete
  • % Complete — Overall completion percentage (Earned / Budget)

Other Sidebar Controls

  • Clear Filters — Remove all active filters and show all records
  • Assign — Reassign your selected records to a different user (you can only reassign records assigned to you)
  • Columns — Show or hide grid columns

Column Header Filtering

Each column header has a filter icon that opens Excel-like filtering options:

  • Text columns — Filter by specific values, contains, starts with, ends with, etc.
  • Numeric columns — Filter by equals, greater than, less than, between, etc.
  • Date columns — Filter by date ranges, before, after, etc.
  • Checkbox list — Select/deselect specific values to show or hide

Column filters can be combined with sidebar filters for precise data views.

Grid Grouping

Group rows by column values to organize activities into collapsible sections. This helps you focus on specific work packages, responsible parties, or other categories.

Using the Group Drop Area

A Group Drop Area appears at the top of the grid. To group your data:

  1. Add grouping — Drag any column header into the Group Drop Area
  2. Multi-level grouping — Drag additional columns to create nested groups
  3. Remove grouping — Drag the column chip out of the drop area, or click its X button
  4. Reorder groups — Drag chips within the drop area to change grouping priority

Working with Groups

  • Expand/Collapse — Click the arrow (▶/▼) next to any group header
  • Group count — Each group header shows the number of activities it contains
  • Filtering + Grouping — Groups respect active filters, showing only matching records

Useful Grouping Columns

Column Use Case
WorkPackage See all activities within each work package
RespParty View activities by responsible party/crew
PhaseCode Organize by project phase
ROCStep Group by workflow step
AssignedTo See who owns which activities

Frozen Columns

Keep important columns visible while scrolling horizontally by freezing them in place. Frozen columns stay pinned on the left side of the grid.

  • Freeze columns — Right-click any column header and select "Freeze Columns to Here". That column and all columns to its left will be frozen.
  • Unfreeze — Right-click any column header and select "Unfreeze All Columns".
  • Choose which columns — Drag columns to reorder them, placing the ones you want frozen on the left side of the grid before freezing.

Your frozen column setting is saved and restored automatically.

Grid Editing

You can only edit records assigned to you. Other users' records appear dimmed and are read-only. Double-click any editable cell to enter edit mode. Press Enter to save or Esc to cancel.

Multi-Select Editing

Select multiple rows using Ctrl+Click or Shift+Click, then use the toolbar percent buttons to update all selected records at once.

Date Field Rules

Field Rules

SchFinish Requires percent to be 100%. Cannot be before SchStart. Auto-sets to today when percent reaches 100%.

Context Menu

Right-click selected rows to access:

  • Delete Row(s) — Delete selected records (ownership verified)
  • Copy Row(s) — Copy to clipboard (visible or all columns)
  • Duplicate Row(s) — Create copies of selected records
  • Export Selected — Export selected rows to Excel

Required Metadata Fields

Nine fields are required for sync. Records missing any of these fields will be highlighted in red and cannot be synced until corrected.

Field Description

Metadata Errors

The footer shows a "Metadata Errors: X" button when records have missing required fields. Click it to filter to only those records so you can fix them.

Syncing Data

SYNC Button

The SYNC operation:

  1. Checks Azure connection (prompts to retry if offline)
  2. Validates all records have required metadata
  3. Pushes your local changes (LocalDirty = 1) to Azure
  4. Pulls updates from other users
  5. Resets LocalDirty flag on synced records
Sync Blocked

Sync will not proceed if you have metadata errors. Fix all required fields first, then try again.

SUBMIT WEEK Button

Use SUBMIT WEEK at the end of each reporting period to:

  1. Create a snapshot of your current data
  2. Archive your progress for the week
  3. Enable comparison with previous weeks

If you work on multiple projects, you'll be prompted to select which project to submit.

Column Layouts

Your column widths, order, and visibility are automatically saved. To customize:

  • Resize columns — Drag the edge of a column header left or right
  • Reorder columns — Drag a column header to a new position
  • Hide/show columns — Click the Columns button in the sidebar

Use Settings (⋮) → Grid Layouts to save named layouts or switch between different column configurations.

Copying Data

MILESTONE provides multiple ways to copy grid data for use in Excel or other applications:

Multi-Cell Copy (Ctrl+C)

Select multiple cells using Ctrl+Click or Shift+Click, then press Ctrl+C. The data is copied in Excel-compatible format (tab-separated columns, newline-separated rows) and pastes directly into Excel cells.

Column Header Copy

Right-click any column header to access copy options:

  • Copy Column w/ Header — Copies the column header and all visible row values (newline-separated)
  • Copy Column w/o Header — Copies only the visible row values without the header

These options copy all visible (filtered) rows in the column, making it easy to extract a single column's data.

Row Copy (Context Menu)

Right-click selected rows to access:

  • Copy (Visible Columns) — Copies selected rows with only visible columns
  • Copy (All Columns) — Copies selected rows with all Activity properties

Multi-Cell Paste (Ctrl+V)

Paste values from the clipboard into selected cells. Values are pasted into the leftmost selected column only.

How it works:

  • Single cell selected — Pastes downward from that cell (one value per row)
  • Multiple cells selected — Pastes into the leftmost column of the selection
  • Multi-column clipboard — Only the first column of clipboard data is used

Validation rules (paste is blocked if any fail):

  • Target column must be editable (not read-only or calculated)
  • You must own all affected records (AssignedTo must match your username)
  • Values must be valid for the column type (numbers, dates, etc.)
Auto-Date Behavior

When pasting to PercentEntry, dates are auto-set just like manual edits: SchStart is set when going from 0% to any value, and SchFinish is set when reaching 100%.

4. Schedule Module

Overview

The Schedule Module compares P6 Primavera schedule data with MILESTONE progress data. Use this module to track schedule variances, forecast upcoming work, and document missed deadlines.

Prerequisites

To see data in the Schedule module, you need both of the following for the same week ending date:

  • P6 schedule data imported via File → Import From P6 File
  • A progress snapshot created via SUBMIT WEEK in the Progress module

If no P6 file has been imported, the header displays "(No schedule loaded)". Without both P6 data and a snapshot for the same week, the grid will be empty.

Screenshot: Schedule module full view (schedule-full-view.png)

Layout

The Schedule view uses a master/detail layout:

  • Master Grid (top) — P6 schedule activities with MS rollup data
  • Detail Grid (bottom) — Individual MILESTONE activities linked to the selected schedule row

Click any row in the master grid to see its linked activities in the detail grid below. Edits to percent, dates, and budget hours in the detail grid automatically recalculate the MS rollup values (MS Start, MS Finish, MS %, MS MHs) shown in the master grid.

Snapshot Data Only

The Schedule module works with weekly snapshot data, not live activity records. Changes made here update the snapshot for the selected week but do not modify the current activity data in the Progress module.

Color Coding

The Schedule module uses color highlighting to draw attention to items needing action:

Color Meaning Action Required
Red Required field is empty Enter the required value (Missed Reason or 3WLA date)
Yellow Discrepancy between P6 and MS data Optional — review with scheduler if reconciliation needed

The status bar shows a legend: red square = required fields, yellow square = discrepancies.

Toolbar

Screenshot: Schedule toolbar (schedule-toolbar.png)

Week Selection

The header displays the current "Week Ending" date. Schedule data is filtered to show snapshots for this week.

Filter Buttons

Filter Shows
Missed Start Activities where P6 planned start has passed but MS hasn't started (or started late)
Missed Finish Activities where P6 planned finish has passed but MS hasn't finished (or finished late)
3WLA Activities in the three-week lookahead window needing forecast dates
Discrepancies Dropdown with options: Actual Start, Actual Finish, MHs, % Complete
Required Fields Activities with missing required values (shows count)

Action Buttons

  • Refresh — Reload schedule data and clear all filters
  • Clear Filters — Remove active filter without reloading data
  • SAVE — Save pending changes (turns red when unsaved changes exist)

Column Header Filtering

Each column header has a filter icon that opens Excel-like filtering options. Use these to filter by specific values, text patterns, numeric ranges, or date ranges. Column filters work alongside the toolbar filter buttons.

Master Grid Columns

P6 Data (Read-Only)

These columns show data imported from P6 Primavera:

  • SchedActNO — P6 activity ID
  • Description — Activity name
  • WbsId — Work breakdown structure code
  • P6 Planned Start/Finish — Original schedule dates
  • P6 Actual Start/Finish — P6 recorded actuals
  • P6 % — P6 percent complete
  • P6 MHs — P6 budget hours

MS Rollups (Calculated)

These columns are calculated from linked MILESTONE activities:

  • MS Start — Earliest start date from linked activities
  • MS Finish — Latest finish date (only if all linked activities are complete)
  • MS % — Weighted average percent complete
  • MS MHs — Sum of budget hours from linked activities

Editable Fields

  • Missed Start Reason — Explanation for missed start deadline
  • Missed Finish Reason — Explanation for missed finish deadline
  • 3WLA Start — Forecast start date
  • 3WLA Finish — Forecast finish date

Detail Grid

Click a master row to view its linked MILESTONE activities. The header shows "Activities for: [SchedActNO] - [Description]".

Editable Columns

Column Rules
% (PercentEntry) 0-100. Changing this auto-adjusts Start/Finish dates.
Budget MHs Editable budget hours for this activity
Start (SchStart) Required when percent > 0. Cannot be future date.
Finish (SchFinish) Required when percent = 100. Cannot be before Start.
Auto-Date Behavior

When you change percent from 0 to any value, Start auto-sets to the week ending date. When you set percent to 100, Finish auto-sets to the week ending date.

Three-Week Lookahead (3WLA)

The three-week lookahead forecasts which activities will start or finish in the next 21 days. This helps with planning and resource allocation.

When 3WLA Dates Are Required

A 3WLA date is required (red highlight) when:

  • P6 planned date is within 21 days of the week ending date, OR
  • P6 planned date has passed but no actual date exists yet

When 3WLA Dates Are Locked

Once an actual date is recorded in MILESTONE:

  • 3WLA Start becomes read-only when MS has an actual start
  • 3WLA Finish becomes read-only when MS has an actual finish

Auto-Clear Behavior

When you record an actual start or finish in the detail grid, the corresponding 3WLA forecast date is automatically cleared (no longer needed).

3WLA Workflow

  1. Click the 3WLA filter button to see activities needing forecasts
  2. Red cells indicate required forecast dates
  3. Enter your forecast dates for 3WLA Start and/or 3WLA Finish
  4. Click SAVE to save your forecasts

Missed Reasons

When a P6 deadline passes and MILESTONE hasn't met it, you must document why.

When Missed Start Reason Is Required

  • P6 planned start date has passed
  • AND MS either never started OR started after the P6 date

When Missed Finish Reason Is Required

  • P6 planned finish date has passed
  • AND MS either never finished OR finished after the P6 date

Auto-Populated Reasons

If MS completed before the P6 date, the system auto-populates:

  • "Started Early" — MS started before P6 planned start
  • "Finished Early" — MS finished before P6 planned finish

Missed Reasons Workflow

  1. Click Missed Start or Missed Finish filter
  2. Red cells indicate rows needing explanation
  3. Enter the reason for the delay
  4. Click SAVE

Discrepancies

Yellow highlighting indicates P6 and MS data don't match. Use the Discrepancies dropdown to filter by type:

Discrepancy Type Meaning
Actual Start P6 and MS have different actual start dates (or one has a date and the other doesn't)
Actual Finish P6 and MS have different actual finish dates
MHs MS budget hours differ from P6 by more than 1%
% Complete P6 percent is higher than MS percent (P6 ahead of reality)

P6 Import

Import schedule data from a P6 Primavera Excel export.

Import Steps

  1. Export from P6 Primavera to Excel (TASK sheet format)
  2. In MILESTONE, go to File → Import From P6 File
  3. Select the Excel file
  4. Choose the week ending date for this schedule
  5. Click Import

P6 Column Mapping

P6 Column MILESTONE Field
task_code SchedActNO
task_name Description
wbs_id WbsId
target_start_date P6_PlannedStart
target_end_date P6_PlannedFinish
act_start_date P6_ActualStart
act_end_date P6_ActualFinish
complete_pct P6_PercentComplete (converted from 0-1 to 0-100)
target_work_qty P6_BudgetMHs
Import Replaces Data

P6 import replaces all existing schedule data. Your 3WLA forecasts and missed reasons are preserved if the SchedActNO still exists in the new import.

P6 Export

Export MILESTONE data back to P6 format for updating Primavera.

Export Steps

  1. Go to File → Export To P6 File
  2. Choose a save location
  3. Select start/finish times (for datetime formatting)
  4. Click Export

Exported Data

The export creates an Excel file with:

  • task_code — SchedActNO
  • status_code — Derived from MS percent (Not Started, In Progress, Complete)
  • act_start_date — MS actual start (if percent > 0)
  • act_end_date — MS actual finish (if percent = 100)
  • complete_pct — MS percent complete

Saving Changes

The SAVE button turns red when you have unsaved changes. Changes that require saving:

  • Missed Start/Finish Reason edits
  • 3WLA Start/Finish date edits
  • Detail grid edits (percent, dates, budget hours)

If you try to leave the Schedule module with unsaved changes, you'll be prompted to save or discard.

Schedule Change Log

When you edit values in the Schedule detail grid (percent, budget hours, start/finish dates), those changes are saved to the weekly snapshot but do not automatically update your live Activities in the Progress module. The Schedule Change Log tracks these edits and lets you optionally apply them to your live Activities.

Accessing the Change Log

Go to Tools → Schedule Change Log to open the dialog.

Change Log Contents

The dialog displays all logged changes with the following columns:

Column Description
Apply (checkbox) Select which changes to apply to live Activities
Week End Date The week ending date when the change was made
UniqueID The activity's unique identifier
Description The activity description for context
Field Which field was changed (PercentEntry, BudgetMHs, SchStart, SchFinish)
Old Value The value before your edit
New Value The value after your edit

Applying Changes

  1. Check the boxes next to the changes you want to apply
  2. Use the header checkbox to select/deselect all
  3. Click Apply Selected
  4. Confirm the action in the warning dialog
Caution

Applying changes will overwrite the current values in your live Activities. This action cannot be undone. The affected records will be marked as LocalDirty and included in your next sync.

Duplicate Changes

If you edited the same field on the same activity multiple times, only the most recent change will be applied. Older duplicate entries are automatically skipped but removed from the log.

Automatic Cleanup

  • Applied changes are removed from the log after successful application
  • Log files older than 30 days are automatically purged on application startup
  • Log files are stored in %LocalAppData%\VANTAGE\Logs\ScheduleChanges\

Keyboard Shortcuts

Shortcut Action
Click cell Edit cell (single-click activates edit mode)
Enter Confirm edit
Esc Cancel edit
Click master row Load detail grid for that activity

5. Work Packages

Overview

The Work Packages module generates professional PDF documents for construction work packages. It combines multiple form templates into a single merged PDF with customizable branding and dynamic content through token replacement.

Screenshot: Work Packages full view (wp-full-view.png)

Key capabilities:

  • Multi-form composition — Combine cover sheets, lists, forms, and grids into one document
  • Token replacement — Dynamic content like project names, dates, and activity data
  • Bulk generation — Generate PDFs for multiple work packages at once
  • Custom branding — Use your own logo or the default Summit logo
  • Reusable templates — Create templates once, use across all projects

Layout

The Work Packages view uses a split-pane design:

  • Left Panel — Three tabs for different functions (Generate, WP Templates, Form Templates)
  • Right Panel — PDF preview of templates

Drag the splitter between panels to resize. Your preference is saved automatically.

Generate Tab

The Generate tab is where you create work package PDFs.

Screenshot: Generate tab (wp-generate-tab.png)

Generation Settings

Field Description
Project Select the project to load its available work packages (required)
Work Packages Check one or more work packages to generate. Use "Select All" or "Clear" buttons for convenience.
WP Template Select which work package template defines the forms to include (required)
PKG Manager Optional. Select a user to populate the {PKGManager} token in documents.
Scheduler Optional. Select a user to populate the {Scheduler} token in documents.
WP Name Pattern Define the output filename pattern using tokens. Example: {WorkPackage}-{Area}
Logo Use "(default)" for the built-in Summit logo, or browse to select a custom logo image
Output Folder Where generated PDFs will be saved (required)
Individual PDFs Check to also save each form as a separate PDF in addition to the merged document
WP Name Pattern

Click the "+ Field" button to insert tokens at the cursor position. Common patterns:

  • {WorkPackage} — Just the work package ID
  • {WorkPackage}-{Area} — Work package with area suffix
  • {Area}-{WorkPackage}-{UDF2} — Custom combination

Generation Workflow

  1. Select a Project to load available work packages
  2. Check the Work Packages you want to generate
  3. Select a WP Template that defines which forms to include
  4. Optionally set PKG Manager and Scheduler for token replacement
  5. Configure the WP Name Pattern for output filenames
  6. Confirm or browse for a Logo
  7. Set the Output Folder
  8. Click GENERATE

The status bar shows progress during generation. When complete, a dialog displays the file paths or any errors encountered.

Output Structure

Generated files are organized as:

{OutputFolder}/{ProjectID}/{WPName}/

Each work package folder contains the merged PDF and optionally individual form PDFs.

WP Templates Tab

Work Package Templates define which forms are included in a work package and their order.

Creating a WP Template

  1. Select "+ Add New" from the WP Template dropdown
  2. Enter a Template Name
  3. Set Expiration Days if different from default

  5. Use ▲/▼ buttons to reorder if needed
  6. Click Save
Form Order

The order of forms in the list determines the page order in the generated PDF. The first form becomes the first page(s).

  • Save — Save changes to the database

6. Progress Books

Overview

Progress Books are printable checklists showing activities grouped by Area, Module, or Drawing. Field crews use these to mark completed progress percentages by hand, which can then be scanned and automatically applied to MILESTONE using the AI Progress Scan feature.

Screenshot: Progress Books full view (pb-full-view.png)

The Progress Books workflow:

  1. Generate — Create printable progress books for field crews
  2. Distribute — Print and give to field crews for the work area
  3. Collect — Retrieve completed books with handwritten progress percentages
  4. Scan — Use AI Progress Scan to extract the percentages from scanned images
  5. Apply — Review and apply the scanned values to your activities

Generating Progress Books

To create a progress book:

  1. Navigate to the PROG BOOKS module
  2. Select a Project from the dropdown
  3. Choose your grouping method (Area, Module, or Drawing)
  4. Select which groups to include
  5. Click Generate to create the PDF

The generated PDF contains:

  • Activity ID — The unique identifier for each activity
  • Description — Activity description for field reference
  • Current % — The current percent complete in MILESTONE
  • New % — Blank column for handwritten progress updates
Print Quality

For best AI scanning results, print progress books on white paper with good contrast. Black and white printing works well.

AI Progress Scan

The AI Progress Scan feature uses AWS Textract to automatically read handwritten progress percentages from scanned progress books. This eliminates manual data entry and reduces transcription errors.

Starting a Scan

  1. Click the AI Scan button in the Progress Books toolbar
  2. In the dialog, click Select Files to choose scanned images
  3. Supported formats: PDF, JPEG, PNG
  4. You can select multiple files at once
  5. Click Start Processing to begin the scan

File Requirements

Requirement Recommendation
Format PDF (best), JPEG, or PNG
Resolution 300 DPI or higher for best accuracy
Orientation Pages should be upright (not rotated)
Handwriting Clear, legible numbers in the "New %" column

Processing Status

During processing, the dialog shows:

  • Progress bar with percentage complete
  • Current page being processed
  • Count of extracted entries

You can click Cancel to stop processing at any time.

Scan Results Dialog

After processing completes, the results dialog displays all extracted entries for review.

Results Grid Columns

Column Description
Apply (checkbox) Select which entries to apply to your activities
Activity ID The extracted activity identifier
Description Activity description (looked up from database)
Current % Current percent complete in MILESTONE
New % The scanned percentage value (editable)
Budget MHs Budget man-hours for the activity, or "NOT FOUND" if the Activity ID isn't in the database
Confidence AI confidence level (percentage)
Status Ready, Warning, or Error indicator

Status Indicators

Status Meaning Action
Ready Entry matched an activity and is ready to apply Review and apply
Warning Low confidence or potential issue (e.g., new % is lower than current %) Verify the scanned value is correct
Error Activity ID not found or invalid percentage Cannot apply — manually enter in Progress module

Selection Buttons

  • Select All — Check all entries regardless of status
  • Select Ready — Check only entries with "Ready" status
  • Clear Selection — Uncheck all entries

Contrast Adjustment

If scan results show incorrect values, you can adjust the image contrast and rescan:

  1. Use the Contrast slider (range 1.0 to 2.0)
  2. Higher values increase contrast for faint handwriting
  3. Click Rescan to reprocess files with the new setting
When to Adjust Contrast

Only adjust contrast if the scan returns incorrect values. The default setting (1.2) works well for most scanned documents. Increase contrast if handwriting is faint or the scan is light.

Filtering Results

Each column header has a filter icon for Excel-like filtering. Use these to:

  • Show only "Warning" status entries that need review
  • Filter to specific Activity IDs
  • Find entries with low confidence scores

Editing Values

You can manually edit the New % column if the scanned value is incorrect:

  1. Double-click the cell to enter edit mode
  2. Type the correct percentage value
  3. Press Enter to confirm

Applying Scanned Progress

Once you've reviewed the scan results:

  1. Use the selection buttons or checkboxes to select entries to apply
  2. Verify that selected entries have correct New % values
  3. Click Apply Selected
  4. Confirm the action in the dialog

Applied changes:

  • Update the PercentEntry field for each selected activity
  • Auto-set SchStart date if changing from 0% (sets to today)
  • Auto-set SchFinish date if changing to 100% (sets to today)
  • Mark records as LocalDirty for sync
Review Before Applying

Always verify the scanned values before applying. Pay special attention to entries with "Warning" status or low confidence scores. Applied changes cannot be automatically undone.

After Applying

After applying scanned progress:

  • The dialog shows a summary of applied changes
  • Switch to the PROGRESS module to review updated records
  • Use the "LocalDirty" filter to see modified records
  • Click SYNC to push changes to Azure
OCR Corrections

The AI scan automatically corrects common OCR errors such as "O" being read as "0". If a "00" is detected, it's automatically interpreted as "100" since no one would write "00%" for progress.
