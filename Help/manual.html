1. Getting Started

What is VANTAGE: Milestone?

VANTAGE: Milestone is a construction activity tracking and work package generation system developed by Summit Industrial. It enables field engineers to track progress on industrial construction activities (Receiving, Fabrication, Assembly, Connections, etc.) and generate work packages for field crews. Features include rules of credit tracking, unit rate management, Three Week Look Ahead reporting, automated progress capture from handwritten field reports, and P6 Primavera integration for schedule management.

Before You Begin

Before using VANTAGE: Milestone:

  • Admin setup required — A system administrator must add your Windows username to the VANTAGE: Milestone user database
  • No login needed — VANTAGE: Milestone automatically detects your Windows username
  • First sync — On first launch, you'll need to sync with the central Azure database to download project data (or import or create activities if none exist yet for your project). If you import or create new records, they will be assigned to you initially — but you can reassign them if desired.

Automatic Updates

VANTAGE: Milestone checks for updates each time it starts. If a newer version is available, the update is downloaded and installed automatically before the application opens. During this process, you'll see status messages on the splash screen.

  • Internet required — If the update server is unreachable (no internet connection), the check is skipped and the app starts normally with the current version.
  • No action needed — Updates are fully automatic. The app will briefly close and reopen with the new version.
  • Your data is safe — Updates only replace application files. Your local database and settings are not affected.

3. Progress Module

Overview

The Progress Module is where you track construction activity completion. This is the primary workspace for field engineers entering daily progress and reviewing assigned work.

Typical Workflow

The standard daily workflow is:

  1. Filter — Use the sidebar to filter to your records or specific criteria
  2. Edit — Update percent complete and other fields in the grid
  3. Sync — Push your changes to Azure and pull updates from others

The Progress view has three main areas:

  • Footer — Record count and metadata error indicator

The summary row updates automatically when you edit values or apply filters, showing totals for the currently visible records only.

VANTAGE: Milestone provides multiple ways to copy grid data for use in Excel or other applications:

Paste values from the clipboard into selected cells. Values are pasted into the leftmost selected column only.

How it works:

  • Single cell selected — Pastes downward from that cell (one value per row)
  • Multiple cells selected — Pastes into the leftmost column of the selection
  • Single value to multiple rows — Copy one cell, select multiple rows with Ctrl+Shift+Down, then Ctrl+V to paste that value into all selected rows (Excel-style fill)
  • Multi-column clipboard — Only the first column of clipboard data is used

Validation rules (paste is blocked if any fail):

  • Target column must be editable (not read-only or calculated)
  • You must own all affected records (AssignedTo must match your username)
  • Values must be valid for the column type (numbers, dates, etc.)
Required Date Fields

When PercentEntry is greater than 0, ActStart is required. When PercentEntry equals 100%, ActFin is also required. Missing required dates show red cell backgrounds and must be filled before syncing.

4. Schedule Module

Overview

The Schedule Module compares P6 Primavera schedule data with VANTAGE: Milestone progress data. Use this module to track schedule variances, forecast upcoming work, and document missed deadlines.

Prerequisites

To see data in the Schedule module, you need both of the following for the same week ending date:

  • P6 schedule data imported via File → Import From P6 File
  • A progress snapshot created via SUBMIT WEEK in the Progress module

If no P6 file has been imported, the header displays "(No schedule loaded)". Without both P6 data and a snapshot for the same week, the grid will be empty.

Layout

The Schedule view uses a master/detail layout:

  • Master Grid (top) — P6 schedule activities with MS rollup data
  • Detail Grid (bottom) — Individual VANTAGE: Milestone activities linked to the selected schedule row

Click any row in the master grid to see its linked activities in the detail grid below. Edits to percent, dates, and budget hours in the detail grid automatically recalculate the rollup values (V-Start, V-Finish, MS %, MS MHs) shown in the master grid.

Snapshot Data Only

The Schedule module works with weekly snapshot data, not live activity records. Changes made here update the snapshot for the selected week but do not modify the current activity data in the Progress module.

Color Coding

The Schedule module uses color highlighting to draw attention to items needing action:

Color Meaning Action Required
Red Required field is empty Enter the required value (Missed Reason or 3WLA date)
Yellow Discrepancy between P6 and MS data Optional — review with scheduler if reconciliation needed

The status bar shows a legend: red square = required fields, yellow square = discrepancies.

These dates are auto-populated when you change PercentEntry: ActStart is set to the WeekEndDate when percent goes from 0 to any positive value, and ActFin is set when percent reaches 100.

Three-Week Lookahead (3WLA)

The three-week lookahead forecasts which activities will start or finish in the next 21 days. This helps with planning and resource allocation.

When 3WLA Dates Are Required

A 3WLA date is required (red highlight) when:

  • P6 planned date is within 21 days of the week ending date, OR
  • P6 planned date has passed but no actual date exists yet

Date Validation

3WLA dates are forecasts — they represent when you expect an activity will start or finish. Dates earlier than the week ending date are not allowed.

If an activity actually started or finished before the week ending date, do not enter a past 3WLA date. Instead, expand the row and edit the detail activities below to set the actual Start or Finish date. This creates the actual date in the system and removes the 3WLA requirement for that activity.

When 3WLA Dates Are Locked

Once an actual date is recorded in VANTAGE: Milestone:

  • 3WLA Start becomes read-only when MS has an actual start
  • 3WLA Finish becomes read-only when MS has an actual finish

Auto-Clear Behavior

When you record an actual start or finish in the detail grid, the corresponding 3WLA forecast date is automatically cleared (no longer needed).

Pre-Population from Activities

3WLA dates are pre-populated from your Activities data when you load the Schedule module. The system finds the earliest PlanStart and latest PlanFin dates across all activities linked to each schedule activity.

Yellow Highlighting

Yellow cells indicate a forecast date that differs from the P6 planned date. This highlights where your forecast diverges from the original schedule. Red cells take priority — if a cell is both required and has a different forecast, it shows red.

3WLA Workflow

  1. Click the 3WLA filter button to see activities needing forecasts
  2. Red cells indicate required forecast dates; yellow cells indicate forecasts that differ from P6
  3. Enter your forecast dates for 3WLA Start and/or 3WLA Finish (must be on or after the week ending date)
  4. Click SAVE to save your forecasts (saved to Activities.PlanStart/PlanFin)

Missed Reasons

When a P6 deadline within the current week passes and VANTAGE: Milestone hasn't met it, you must document why.

When Missed Start Reason Is Required

  • P6 planned start date is within the current week (WeekEndDate - 7 days to WeekEndDate)
  • AND MS either never started OR started after the P6 date

When Missed Finish Reason Is Required

  • P6 planned finish date is within the current week (WeekEndDate - 7 days to WeekEndDate)
  • AND MS either never finished OR finished after the P6 date
Session-Only Persistence

Missed reasons are stored in the Schedule data and persist within your session, but are cleared when you import a new P6 file. This is expected behavior since P6 imports represent fresh schedule data.

Auto-Populated Reasons

If MS completed before the P6 date, the system auto-populates:

  • "Started Early" — MS started before P6 planned start
  • "Finished Early" — MS finished before P6 planned finish

Missed Reasons Workflow

  1. Click Missed Start or Missed Finish filter
  2. Red cells indicate rows needing explanation
  3. Enter the reason for the delay
  4. Click SAVE

Discrepancies

Yellow highlighting indicates P6 and VANTAGE data don't match (legend shows "Vtg ≠ P6"). Use the Discrepancies dropdown to filter by type:

Discrepancy Type Meaning
% Complete P6 percent is higher than VANTAGE percent (P6 ahead of reality)
3WLA Finish 3WLA forecast finish date differs from P6 planned finish date
3WLA Start 3WLA forecast start date differs from P6 planned start date
Actual Finish P6 and VANTAGE have different actual finish dates
Actual Start P6 and VANTAGE have different actual start dates (or one has a date and the other doesn't)
MHs VANTAGE budget hours differ from P6 by more than 1%

Import schedule data from a P6 Primavera Excel export.

  2. In VANTAGE: Milestone, go to File → Import From P6 File
  3. Select the Excel file
  4. Choose the week ending date for this schedule
  5. Select which projects this schedule covers (the project list shows only projects for which you have submitted snapshots)
  6. Click Import

The project selection list is populated from your submitted snapshots. If a project doesn't appear, you need to Submit Week for that project in the Progress module first.

Import Replaces Data

P6 import replaces all existing schedule data. Your 3WLA forecast dates are preserved via the Activities PlanStart/PlanFin fields and will reload automatically. Missed reasons are cleared on import since they apply to the previous schedule data.

UDF Column Mapping

The Schedule module supports five custom UDF columns (SchedUDF1-5) that can be mapped to any P6 user-defined fields in your schedule export. This allows you to import custom P6 data into VANTAGE: Milestone.

Configuring UDF Mappings

  1. Go to Tools → Schedule UDF Mapping...
  2. For each UDF column you want to use:
    • Primary Header — The P6 technical column name from row 1 of the export (e.g., udf_text1)
    • Secondary Header — The P6 display name from row 2 (e.g., Contractor Code)
    • Display Name — Your preferred column header in VANTAGE (defaults to SchedUDF1 if blank)
  3. Click Save

A UDF column is considered configured if it has either a Primary Header or Secondary Header filled in.

How Mapping Works

During P6 import, VANTAGE: Milestone searches for each configured UDF column:

  1. First tries to match the Primary Header (row 1, exact match)
  2. Falls back to Secondary Header (row 2, normalized) if primary not found
  3. If neither is found, the UDF column remains empty

  3. Your visibility choices are saved and persist across sessions

UDF values are imported as text and are read-only. When you change UDF mappings and re-import a schedule, any UDF columns not in the new mapping are cleared to prevent stale data.

6. Progress Books

Overview

Progress Books are printable checklists showing activities grouped by user-selected columns (e.g., Area, Module, or Drawing). The Progress Books workflow:

  2. Distribute — Print and give to field crews for the work area
  3. Collect — Retrieve completed books with handwritten progress percentages

  5. Apply — Review and apply the scanned values to your activities

Layout

Uploads selected snapshots to the VANTAGE_global_ProgressLog table on Azure for company-wide historical tracking.

  • Select one or more snapshot groups, then click Upload to Progress Log
  • Column mapping is driven by the VMS_ColumnMappings table (Milestone column names are mapped to their Azure/ProgressLog equivalents)
  • Calculated fields are computed at upload time:
    • Status — "Not Started" (0%), "In Progress" (1-99%), or "Complete" (100%) based on PercentEntry
    • EarnMHsCalc — BudgetMHs x PercentEntry / 100
    • ClientEquivEarnQTY — (PercentEntry / 100) x ClientEquivQty
  • UserID is set to the admin performing the upload
  • Timestamp is set to the upload time (identical for all records in a batch)
  • After upload, AzureUploadUtcDate is updated on VMS_Activities for all uploaded records so users can see when their data was last uploaded
  • If records for the same Project/Week have already been uploaded, a duplicate warning is shown before proceeding
  • Each upload is tracked in VMS_ProgressLogUploads so it can be found and deleted later via Manage Progress Log

  • Use Select All / Select None buttons for bulk selection

Importing Legacy Uploads (REFRESH)

  • The REFRESH dropdown scans VANTAGE_global_ProgressLog for uploads not yet tracked in this dialog
  • Choose a time range: Last 7 days, Last 30 days, or All time
  • A confirmation dialog appears before scanning begins
  • New batches found are added to the tracking table so they can be managed (viewed/deleted) from this dialog
  • For imported batches: Username = admin who ran REFRESH, Uploaded By = original uploader from OldVantage
  • Completion message shows how many new batches were imported and elapsed time

  • Select one or more batches using the checkboxes, then click Delete Selected
  • A confirmation dialog requires typing DELETE to proceed (prevents accidental deletion)
  • This permanently removes the matching records from both VANTAGE_global_ProgressLog and the tracking table
  • Use this when progress data needs to be corrected and re-uploaded for a given week

Database Sync

All users sync their local data with the central Azure database using the SYNC button in the Progress module (see Syncing Data). Key points for administrators:

  • Azure is the source of truth — Local SQLite databases are caches that can be deleted and re-synced at any time
  • Ownership — Users can only edit activities assigned to them. Admins bypass ownership checks and can edit any record.
  • Pull-only fields — Some fields (e.g., AzureUploadUtcDate) are set by admins on Azure and received by users on pull, but never overwritten by user push operations
  • Clear Local Activities — Available from the Settings menu to wipe and re-download all local data from Azure

  • Azure - Names used in the central database sync

Troubleshooting

Cannot connect to Azure database

If the connection error dialog appears on startup, your network may not be whitelisted in the Azure firewall. The dialog will detect and display your public IP address. Click COPY to copy it to your clipboard, then email your administrator and ask them to add your IP to the Azure SQL firewall rules. Once whitelisted, click RETRY to connect. You can also click WORK OFFLINE to use the app with your local data until the issue is resolved.

Windows SmartScreen blocks the application

When running VANTAGE for the first time, Windows SmartScreen may display a warning because the application is not digitally signed. To proceed:

  • Click More info in the SmartScreen dialog, then click Run anyway
  • Alternatively, right-click the exe file, select Properties, check Unblock at the bottom, and click OK

Application won't start after an update

If the app fails to launch after an auto-update, the encrypted configuration file may be missing. Check that appsettings.enc exists in your install directory (%LOCALAPPDATA%\VANTAGE\App\). If it is missing, re-run the installer to perform a fresh install.
